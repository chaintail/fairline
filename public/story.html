FairLine

A model-vs-market pricing oracle for in-play soccer — reprices continuously from real match state, publishes fair odds as signed data, logs its own edge against the market.

“It is positioned as a pricing oracle, not a quoting bot”

Ed25519-signed feed Dixon-Coles in-play Poisson model real TxLINE mainnet feed MCP + llms.txt for agents

Why it's different

Odds you can check, not odds you take on faith

◎

Checkable, not asserted

Most in-play odds products are black boxes. FairLine publishes its actual model probabilities right next to the market's, so its "edge" is something you can check yourself, tick by tick — not a claim you have to trust.

🔏

Signed, not just stated

Every published tick is an Ed25519-signed envelope. Anyone — human or agent — can verify any price against the feed's public key, offline, with no API trust required.

≡

One code path, honestly labeled

The engine runs identically over a labeled synthetic replay (for demos) and the real TxLINE feed (for the live final). Same reducers, same model — never blended, always labeled which one you're looking at.

How it works

Score and clock in, signed fair prices out

01 TxLINE score / clock / odds state official match state + de-margined market consensus
→
02 Dixon-Coles-corrected in-play Poisson model time-varying scoring intensity, red-card adjustments
→
03 Fair prices — 1X2 + totals model probabilities converted to fair odds
→
04 Signed envelope Ed25519, canonical serialization, verifiable offline
→
05 SSE feed + edge log public stream, downloadable JSON/CSV history
